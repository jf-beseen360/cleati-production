🚀 PROCESSUS D'EMBARQUEMENT EN 5 ÉTAPES

ÉTAPE 1: Exécuter le Script de Configuration Automatique
═════════════════════════════════════════════════════════

$ bash QUICK_GITHUB_SETUP.sh

Cela va:
  ✓ Configurer Git avec votre email
  ✓ Initialiser le dépôt local
  ✓ Ajouter tous les fichiers
  ✓ Créer le commit initial
  ✓ Ajouter la remote GitHub


ÉTAPE 2: Pousser le Code vers GitHub
════════════════════════════════════

$ git push -u origin main

Cela va:
  ✓ Pousser tous les fichiers
  ✓ Créer la branche main sur GitHub
  ✓ Déclencher le pipeline CI/CD


ÉTAPE 3: Créer la Branche Develop
════════════════════════════════

$ git checkout -b develop
$ git push -u origin develop

Cela va:
  ✓ Créer la branche develop (pour staging)
  ✓ Pousser vers GitHub


ÉTAPE 4: Vérifier sur GitHub
═══════════════════════════

1. Aller à: https://github.com/jf-beseen360/cleati-production
2. Vérifier que tous les fichiers sont présents
3. Aller à l'onglet "Actions"
4. Vérifier que les workflows s'exécutent

Attendez ~20 minutes pour le pipeline complet:
  🔵 Tests (2 min)
  🔵 Docker Build (5-10 min)
  🔵 Security Scan (2 min)
  🟢 Tous les jobs devraient passer


ÉTAPE 5: Configurer les Paramètres GitHub (Optionnel)
════════════════════════════════════════════════════

1. Settings → Branches
   → Ajouter les règles de protection pour main
   → Ajouter les règles de protection pour develop

2. Settings → Secrets and variables → Actions
   → Ajouter les secrets de déploiement (optionnel)

3. Settings → Actions → General
   → Configurer les permissions
